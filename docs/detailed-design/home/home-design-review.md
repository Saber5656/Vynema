### 4.2 Mux コストの精査

設計書のMux見積もり:

| 項目 | 設計書の単価 | 設計書の想定量 | 月額 |
|------|-----------|-------------|------|
| Encoding | $0.015/分 | 100分 | $1.50 |
| Storage | $0.007/GB/月 | 10GB | $0.07 |
| Streaming | $0.006/分視聴 | 2,000分 | $12.00 |
| **合計** | | | **$13.57** |

**reviewer の指摘:**

1. **Storage単価の確認**: $0.007/GB/月は適切だが、ABR用のマルチ解像度ファイルが保存されるため、1動画のストレージは生のMP4より2-3倍になる。10GBは100本×100MBの計算だが、ABR後は10本×300MB程度で3GB/月、動画が蓄積すると半年で約20GB → $0.14/月。**大きな影響はない**
2. **Streaming Delivery単価**: $0.006/分は妥当。ただし**バイラル動画が発生した場合のリスク**が設計書に明記されていない
3. **月10,000視聴超でMux Dataが有料化**する点がPMの注意点として指摘済み

**結論**: Muxコストの見積もりは概ね妥当。ただし以下のセーフガードを推奨:
- Muxダッシュボードでの利用量アラート設定
- 月間配信分数の上限モニタリング

### 4.3 AI生成コストの精査

| API | 設計書の単価 | 月30本時 | 月100本時 |
|-----|-----------|---------|----------|
| Runway Gen-4 Turbo | ~$0.50/10秒 | $15 | $50 |
| Veo 3.1 Fast | ~$0.30/10秒 | $9 | $30 |

**reviewer の指摘:**

1. **生成失敗時のリトライコスト**: 設計書に言及なし。品質チェック不合格 → 再生成で実質コスト1.3-1.5倍になる可能性
2. **動画長の前提**: 10秒動画の前提だが、UX設計のVideoCardに `[12:34]` のような長い動画も想定されている。1分動画なら$3/本（Runway）
3. **月30本推奨は妥当**: $15-25/月に収まる

**推奨**: MVP段階では以下の簡易実装を提案:
- `tab=trending` → 「今日急上昇中の動画」固定テキスト
- `tab=new` → 「新着動画」固定テキスト
- `tab=quality` → 「品質スコアが高い人気動画」固定テキスト
- パーソナライズド理由は将来フェーズ

##### [CONSISTENCY-ISSUE-2] Hero Section の priority 指定漏れ（低）

**問題**: パフォーマンス設計でAbove the foldの4枚に`priority={true}`を指定しているが、Hero Sectionのサムネイル（最もLCPになりやすい要素）への`priority`指定が明示されていない。

**推奨**: Hero Section のサムネイルにも `priority={true}` を追加。

## 7. 独自性・差別化の評価（ポジティブフィードバック）

以下の設計は特に優れており、YouTubeとの明確な差別化になる:

### 7.1 Quality Score システム

- 再生数に代わる多軸評価（技術品質/いいね率/視聴完了率/プロンプト創造性/コメント質）は、AI生成コンテンツならではの画期的な指標
- VideoCardのコンパクト表示（★ 4.2）から動画再生ページの詳細表示（5軸バー）まで、段階的な情報開示が優れている
- MVPでは `likeRatio × 50 + completionRate × 50` の簡易算出から始め、段階的に拡張する計画は現実的

### 7.2 ムードブラウジング

- カテゴリ（音楽/ゲーム等）ではなく「ムード/雰囲気」（Calm/Energetic/Dreamy等）でコンテンツを探索するUXは独創的
- グラデーション背景のMoodCardのビジュアルデザインが魅力的
- `Video.moods` String[] + GINインデックスでDB対応も適切

### 7.3 おすすめ理由の透明表示

- YouTubeの「なぜこの動画がおすすめされたか不透明」という課題への直接的な解答
- AI専用プラットフォームだからこそ、推薦アルゴリズムの全面開示が可能という構造的優位性を活かしている

### 7.4 ホバープレビュー

- Mux Animated Thumbnail API (`animated.gif`) を活用した低コスト実装
- 300msディレイ → フェードインの段階的表示で、意図しないプレビュー表示を防止

## 9. MVP着手判定

### 判定: **着手可（条件付き）**

4つの詳細設計ドキュメントは、個人開発のMVPとして十分な品質と整合性を持っている。以下の条件を満たした上で実装フェーズに移行することを推奨:

1. **重要度「中」の指摘6件のうち、少なくとも以下3件を対応**:
   - US-ISSUE-1: 検索APIの最低限の設計追加
   - A11Y-ISSUE-1: AIBadgeの aria-label 追加
   - SEC-ISSUE-1: CSPヘッダーの定義

2. **コスト管理ルールの確定**:
   - 月間動画生成本数の上限（推奨: 30-50本）
   - Mux配信量のモニタリング頻度

3. **残りの指摘は実装フェーズ中に段階的に対応可能**
