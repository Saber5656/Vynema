## 1. ユーザーストーリー（視聴者 = 人間ユーザー）

### 1.1 アカウント管理

| ID | ユーザーストーリー | 優先度 | 受入基準 |
|----|-------------------|--------|----------|
| US-001 | 視聴者として、メールアドレスまたはソーシャルログインでアカウントを作成したい。動画の保存やコメントをするために。 | Must | Clerk経由でGoogle/GitHub OAuthサインアップが可能。メール認証必須。 |
| US-002 | 視聴者として、ログインせずに動画を視聴したい。気軽にコンテンツを楽しむために。 | Must | 未ログインでも動画一覧閲覧・再生が可能。 |
| US-003 | 視聴者として、プロフィール（表示名・アバター）を設定したい。コミュニティ内での識別のために。 | Should | プロフィール編集画面から表示名・アバター画像の変更が可能。 |
| US-004 | 視聴者として、アカウントを削除したい。サービスから離れたいときのために。 | Must | アカウント削除によりユーザーデータが適切に処理される（GDPR/個人情報保護法対応）。 |

### 1.2 動画視聴

| ID | ユーザーストーリー | 優先度 | 受入基準 |
|----|-------------------|--------|----------|
| US-010 | 視聴者として、トップページでおすすめ動画を閲覧したい。新しいAI生成コンテンツを発見するために。 | Must | トップページにパーソナライズされた/人気の動画一覧が表示される。 |
| US-011 | 視聴者として、動画をスムーズに再生したい。快適な視聴体験のために。 | Must | Muxによるアダプティブビットレート対応。再生開始まで3秒以内。途中バッファリング最小限。 |
| US-012 | 視聴者として、動画の再生速度を変更したい。効率的に視聴するために。 | Should | 0.25x〜2x の速度調整が可能。 |
| US-013 | 視聴者として、動画の画質を手動選択したい。通信環境に応じた視聴のために。 | Should | 360p/480p/720p/1080p から選択可能（利用可能な場合）。 |
| US-014 | 視聴者として、動画をフルスクリーンで視聴したい。没入感のある視聴のために。 | Must | フルスクリーンモード、ピクチャーインピクチャー対応。 |
| US-015 | 視聴者として、動画にキーボードショートカットで操作したい。効率的な操作のために。 | Should | スペース(再生/停止)、矢印キー(シーク)、F(フルスクリーン)、M(ミュート)。 |
| US-016 | 視聴者として、AI生成コンテンツであることを明確に認識したい。情報の透明性のために。 | Must | 動画プレーヤー上および動画情報欄に「AI生成コンテンツ」ラベルを常時表示。 |

### 1.3 検索・発見

| ID | ユーザーストーリー | 優先度 | 受入基準 |
|----|-------------------|--------|----------|
| US-020 | 視聴者として、キーワードで動画を検索したい。見たいコンテンツを見つけるために。 | Must | タイトル、説明文、タグで全文検索可能。検索結果は関連度順。 |
| US-021 | 視聴者として、カテゴリやタグで動画をフィルタリングしたい。ジャンル別に探索するために。 | Should | カテゴリ（教育、エンタメ、音楽、アート等）、AI生成モデル別フィルター。 |
| US-022 | 視聴者として、トレンド動画を閲覧したい。人気コンテンツを把握するために。 | Should | トレンドページに再生回数・いいね数ベースのランキング。 |
| US-023 | 視聴者として、視聴履歴に基づくレコメンドを受けたい。好みに合ったコンテンツを発見するために。 | Could | 視聴履歴・いいね履歴に基づくパーソナライズドレコメンド。 |

### 1.4 ソーシャル機能

| ID | ユーザーストーリー | 優先度 | 受入基準 |
|----|-------------------|--------|----------|
| US-030 | 視聴者として、動画にコメントしたい。感想や意見を共有するために。 | Must | ログインユーザー（Clerk認証済み）がテキストコメントを投稿可能。Supabase Realtimeで他ユーザーのコメントがリアルタイム反映。 |
| US-031 | 視聴者として、コメントに返信したい。議論に参加するために。 | Should | スレッド形式の返信が可能。リアルタイム更新対応。 |
| US-032 | 視聴者として、動画にいいね/低評価をしたい。コンテンツの評価を表明するために。 | Must | いいね/低評価ボタン。1ユーザー1動画に対し1回。取消可能。Supabase Realtimeでカウントがリアルタイム更新。 |
| US-033 | 視聴者として、動画をお気に入りリストに保存したい。後で見返すために。 | Should | お気に入りリストへの追加/削除。マイページから一覧表示。 |
| US-034 | 視聴者として、動画を共有（URL コピー、SNS共有）したい。他の人にも見てもらうために。 | Should | 共有ボタンからURL コピー、Twitter/X、Facebook等へのシェア。 |
| US-035 | 視聴者として、不適切なコメントを報告したい。コミュニティの健全性のために。 | Must | 報告ボタンからスパム/ハラスメント等のカテゴリを選択して通報。 |

### 1.5 AIチャンネル

| ID | ユーザーストーリー | 優先度 | 受入基準 |
|----|-------------------|--------|----------|
| US-040 | 視聴者として、AIチャンネル（AI投稿者）のページを閲覧したい。特定のAI投稿者のコンテンツをまとめて見るために。 | Must | AIチャンネルページに投稿者情報、投稿動画一覧、統計情報が表示。 |
| US-041 | 視聴者として、AIチャンネルをフォロー（チャンネル登録）したい。新着動画を追いかけるために。 | Should | フォローボタン。フォロー中チャンネルの新着動画がフィードに表示。 |
| US-042 | 視聴者として、AIがどのようなモデル/手法で動画を生成したかを知りたい。技術的な興味のために。 | Should | 動画情報欄に使用AIモデル、プロンプト概要等のメタデータ表示。 |

## 2. AI投稿者のユースケース（システムとしてのAI動画生成パイプライン）

### 2.1 動画生成パイプライン概要

```
[トリガー] → [プロンプト生成] → [AI動画生成API呼出] → [コンテンツ安全性チェック] → [Muxアップロード] → [メタデータ生成] → [Supabase DB登録・公開]
```

**確定技術スタック対応:**
- 認証: Clerk（Google/GitHub OAuth）
- 動画配信: Mux（アップロード・トランスコーディング・ABRストリーミング・CDN一括管理）
- リアルタイム機能: Supabase Realtime（コメント・いいねのリアルタイム反映）

### 2.2 ユースケース一覧

| ID     | ユースケース           | アクター                 | 説明                                                             |
| ------ | ---------------- | -------------------- | -------------------------------------------------------------- |
| UC-001 | スケジュールベース動画生成    | システム（Cron/スケジューラー）   | 定期スケジュール（例: 毎日、毎時間）に基づいて自動的に動画生成パイプラインを起動する。                   |
| UC-002 | トレンドベース動画生成      | システム（トレンド分析エンジン）     | トレンドトピック・ニュースを分析し、それに基づいたプロンプトを生成して動画を作成する。                    |
| UC-003 | ユーザーリクエストベース動画生成 | 視聴者（トリガー）→ システム（実行）  | 視聴者が「こんな動画が見たい」とリクエストし、AIがそれを元に動画を生成する（将来機能）。                  |
| UC-004 | 動画プロンプト生成        | LLM（GPT-4/Claude等）   | テーマ/カテゴリに基づき、動画生成AIに渡すプロンプトをテキストAIが生成する。                       |
| UC-005 | 動画本体の生成          | 動画生成AI（Runway/Sora等） | プロンプトをもとに動画ファイル（MP4等）を生成する。品質チェック含む。                           |
| UC-006 | サムネイル自動生成        | システム（FFmpeg/AI画像生成）  | 動画の代表フレーム抽出、またはAI画像生成によるサムネイル作成。                               |
| UC-007 | メタデータ自動生成        | LLM                  | タイトル、説明文、タグ、カテゴリの自動生成。SEO最適化含む。                                |
| UC-008 | コンテンツ安全性チェック     | コンテンツモデレーションAPI      | 生成された動画が安全基準を満たしているか自動検証。NSFW/暴力コンテンツの検出。                      |
| UC-009 | 動画トランスコーディング     | Mux（フルマネージド）         | Muxへの動画アップロードにより自動トランスコーディング。複数解像度・ABR対応はMux側で処理。              |
| UC-010 | 動画アップロード・公開      | システム                 | 生成動画をMuxにアップロードし、Playback IDを取得。Supabaseデータベースにメタデータを登録して公開する。 |
| UC-011 | 生成失敗時のリトライ       | システム                 | API呼出失敗、品質チェック不合格時の自動リトライ。フォールバックAPI切替。                        |

### 2.3 AI投稿者（チャンネル）のモデル

AIチャンネルは「テーマ/ジャンル × AI生成スタイル」で定義される仮想的な投稿者:

| チャンネル例 | テーマ | AI生成スタイル |
|-------------|--------|---------------|
| AI Nature | 自然・風景 | フォトリアル |
| AI Abstract Art | 抽象アート | アーティスティック |
| AI News Digest | ニュースダイジェスト | インフォグラフィック |
| AI Music Visuals | 音楽ビジュアル | ミュージックビデオ風 |
| AI Stories | ショートストーリー | アニメーション |

## 3. 非機能要件

### 3.1 パフォーマンス

| ID | 要件 | 目標値 | 計測方法 |
|----|------|--------|----------|
| NFR-P01 | 初期ページロード時間 | LCP 2.5秒以内 | Lighthouse / Web Vitals |
| NFR-P02 | 動画再生開始時間 | 3秒以内 | カスタムメトリクス |
| NFR-P03 | 検索結果表示時間 | 1秒以内 | API レスポンスタイム |
| NFR-P04 | API レスポンスタイム（一般） | p95 で 500ms以内 | サーバーサイドモニタリング |
| NFR-P05 | 同時接続ユーザー数 | MVP: 100、将来: 10,000+ | 負荷テスト |
| NFR-P06 | 動画ストリーミング品質 | Mux ABR対応、バッファリング率 1% 以下 | Mux Data / プレーヤーメトリクス |
| NFR-P07 | ページ間遷移速度 | 300ms以内（SPA遷移） | Navigation Timing API |

### 3.2 アクセシビリティ

| ID | 要件 | 基準 |
|----|------|------|
| NFR-A01 | WCAG 2.1 Level AA 準拠 | 色コントラスト比 4.5:1以上、フォーカス管理、ARIAラベル |
| NFR-A02 | キーボードナビゲーション | すべてのインタラクティブ要素にキーボードでアクセス可能 |
| NFR-A03 | スクリーンリーダー対応 | 動画プレーヤー、ナビゲーション、フォームがスクリーンリーダーで操作可能 |
| NFR-A04 | 字幕/キャプション | AI生成動画に自動字幕を付与（音声がある場合） |
| NFR-A05 | レスポンシブデザイン | モバイル（320px〜）/ タブレット / デスクトップ対応 |
| NFR-A06 | 減モーション対応 | prefers-reduced-motion メディアクエリ対応 |

### 3.3 セキュリティ

| ID | 要件 | 対策 |
|----|------|------|
| NFR-S01 | 認証・認可 | Clerkによるセッション管理。Google/GitHub OAuthフロー。Clerkのセキュリティ基盤に依拠。 |
| NFR-S02 | CSRF 対策 | CSRFトークンの実装。SameSite Cookie属性設定。 |
| NFR-S03 | XSS 対策 | コメント等のユーザー入力のサニタイズ。CSP ヘッダー設定。 |
| NFR-S04 | SQLインジェクション対策 | ORM 使用によるパラメタライズドクエリ。 |
| NFR-S05 | レート制限 | API エンドポイントにレート制限。コメント投稿にスパム防止。 |
| NFR-S06 | HTTPS 強制 | すべての通信をHTTPS化。HSTS ヘッダー設定。 |
| NFR-S07 | 動画コンテンツの完全性 | AI生成パイプラインのみが動画を投稿可能な設計。人間による直接アップロードを技術的に防止。 |
| NFR-S08 | API キー管理 | AI生成API等の外部APIキーはサーバーサイドのみで管理。環境変数/Secrets Manager使用。 |
| NFR-S09 | ユーザーデータ保護 | 個人情報の暗号化保存。ログからの個人情報除外。 |
| NFR-S10 | コメントモデレーション | 自動スパム検出 + 手動報告システム。 |

### 3.4 可用性・運用

| ID | 要件 | 目標値 |
|----|------|--------|
| NFR-O01 | 稼働率 | 99.5%以上（MVP）、99.9%以上（将来） |
| NFR-O02 | バックアップ | データベース: 日次バックアップ、動画ファイル: 冗長ストレージ |
| NFR-O03 | モニタリング | エラートラッキング、パフォーマンスモニタリング、アラート設定 |
| NFR-O04 | ログ | 構造化ログ出力。動画生成パイプラインの追跡可能性確保。 |

## 4. 法的・倫理的考慮事項

### 4.1 AI生成コンテンツの透明性（最重要）

| 項目 | 対応方針 |
|------|----------|
| **表示義務** | すべての動画に「AI生成コンテンツ」であることを明確に表示する。EU AI Act（2025年施行）、日本のAIガイドライン等を考慮。 |
| **表示方法** | ① 動画プレーヤー上のウォーターマーク/バッジ ② 動画情報欄の明示的ラベル ③ メタデータ（C2PA/Content Credentials 対応を検討） |
| **使用AIモデルの開示** | 各動画がどのAIモデルで生成されたか、プロンプトの概要を公開する。 |
| **ディープフェイク防止** | 実在の人物に類似するコンテンツの生成を禁止するポリシーを設定。コンテンツモデレーションで検出。 |

### 4.2 著作権・知的財産権

| 項目 | 対応方針 |
|------|----------|
| **AI生成物の著作権** | AI生成コンテンツの著作権帰属はサービス運営者に帰属する旨を利用規約に明記。（各国の法律状況に応じて更新が必要） |
| **学習データの権利** | 使用するAI生成APIのデータソースに関する情報を確認し、著作権侵害リスクを評価。 |
| **ユーザーコメントの権利** | ユーザーが投稿したコメントの権利はユーザーに帰属。サービスでの表示・利用に関するライセンスを利用規約で取得。 |
| **音楽・音声の権利** | AI生成動画に音楽・音声が含まれる場合、ロイヤリティフリー素材使用 or AI生成音楽使用を徹底。 |
| **DMCA/削除請求対応** | 著作権侵害の申告を受け付けるプロセスを設置（DMCA テイクダウン手続き）。 |

### 4.3 プライバシー・データ保護

| 項目 | 対応方針 |
|------|----------|
| **個人情報保護法/GDPR** | ユーザーデータの収集・利用目的を明示。データ削除要求への対応。Cookie同意バナー。 |
| **視聴データの利用** | 視聴履歴、いいね履歴等の利用範囲をプライバシーポリシーに明記。 |
| **アナリティクス** | プライバシーに配慮したアナリティクス（Plausible、Umami等）の使用を推奨。 |
| **子供の保護** | 13歳未満の利用に関するCOPPA対応の検討。年齢確認の必要性評価。 |

### 4.4 倫理的考慮

| 項目 | 対応方針 |
|------|----------|
| **コンテンツポリシー** | AI生成動画のコンテンツガイドラインを策定。暴力、ヘイト、NSFW等の禁止カテゴリ。 |
| **バイアス対策** | AI生成コンテンツにおけるバイアス（人種、性別、文化等）のモニタリングと是正。 |
| **誤情報防止** | ニュース/情報系コンテンツにおけるファクトチェックの仕組み。「これはAI生成の創作コンテンツです」の免責表示。 |
| **環境負荷** | AI動画生成のエネルギー消費に関する情報開示を検討。 |
| **中毒性設計の回避** | 無限スクロール・自動再生等の中毒性設計の節度ある実装。視聴時間リマインダー等の導入を検討。 |

### 4.5 利用規約に含めるべき主要条項

1. サービスの性質（すべての動画はAIによって生成されたものである）
2. ユーザーアカウントの責任範囲
3. AI生成コンテンツの著作権帰属
4. 禁止行為（スパムコメント、ハラスメント、違法コンテンツの報告悪用等）
5. コンテンツモデレーションポリシー
6. プライバシーポリシーへのリンク
7. 免責事項（AI生成コンテンツの正確性・完全性）
8. サービスの変更・終了に関する条項
9. 準拠法・紛争解決

## 5. 品質基準チェックリスト

### 5.1 機能品質

- [ ] すべてのユーザーストーリー（US-001〜US-042）が実装されている（優先度 Must は必須）
- [ ] 動画再生がすべての主要ブラウザ（Chrome, Firefox, Safari, Edge）で動作する
- [ ] モバイル端末（iOS Safari, Android Chrome）で正常に動作する
- [ ] 検索機能が日本語・英語の両方で正しく動作する
- [ ] コメント投稿・表示が正常に動作する
- [ ] いいね/低評価の集計が正確である
- [ ] AIチャンネルページが正しく表示される

### 5.2 AI生成パイプライン品質

- [ ] 動画生成パイプラインが自動的に稼働する
- [ ] 生成失敗時のリトライ・フォールバックが機能する
- [ ] 生成された動画がコンテンツ安全性チェックを通過している
- [ ] メタデータ（タイトル、説明文、タグ）が適切に自動生成されている
- [ ] サムネイルが自動生成されている
- [ ] Muxによる動画トランスコーディング・配信が正常に動作している
- [ ] 「AI生成コンテンツ」ラベルがすべての動画に付与されている

### 5.3 パフォーマンス品質

- [ ] Lighthouse パフォーマンススコア 90 以上
- [ ] Core Web Vitals（LCP < 2.5s, FID < 100ms, CLS < 0.1）達成
- [ ] 動画再生開始が3秒以内
- [ ] API レスポンスタイム p95 < 500ms

### 5.4 セキュリティ品質

- [ ] OWASP Top 10 の脆弱性が対策されている
- [ ] 人間による動画直接アップロードが技術的に不可能な設計になっている
- [ ] すべての外部APIキーがサーバーサイドで安全に管理されている
- [ ] ユーザー入力がすべてサニタイズされている
- [ ] HTTPS がすべてのページで強制されている
- [ ] レート制限が適切に設定されている

### 5.5 アクセシビリティ品質

- [ ] WCAG 2.1 Level AA のすべてのチェック項目をパス
- [ ] キーボードのみですべての操作が可能
- [ ] スクリーンリーダーでの操作テスト完了
- [ ] 色覚異常ユーザーが識別できるUIカラー設計

### 5.6 法的コンプライアンス品質

- [ ] AI生成コンテンツの表示義務を満たしている（EU AI Act 対応）
- [ ] 利用規約が策定され、AI生成コンテンツの著作権帰属が明記されている
- [ ] プライバシーポリシーが策定されている
- [ ] Cookie 同意バナーが実装されている
- [ ] DMCA テイクダウン手続きが設置されている
