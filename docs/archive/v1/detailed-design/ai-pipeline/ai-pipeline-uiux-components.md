## 0. 概要

AI Theater の AI動画生成パイプラインの管理ダッシュボード (`/admin/pipeline`) について、
Runway・BullMQ Dashboard・CI/CD パイプライン UI 等の競合分析をベースに
AI Theater 独自の「AIエージェント運用管理」体験を設計する。

**重要な前提**: AI Theater の動画生成パイプラインは **人間が直接操作するものではなく、
システムが自動実行する**。本 UI は **運用者（project-owner）が状態監視・異常対応・コスト管理** を行うための
管理ダッシュボードである。一般視聴者には公開しない。

### 対象ユースケース（user-requirements-usecases.md 参照）

| UC ID | ユースケース | 本設計での対応 |
|-------|------------|--------------|
| UC-001 | スケジュールベース動画生成 | スケジュール管理UI |
| UC-004 | 動画プロンプト生成 | プロンプトテンプレート管理UI |
| UC-005 | 動画本体の生成 | 生成キュー・進捗表示 |
| UC-008 | コンテンツ安全性チェック | モデレーション結果表示 |
| UC-010 | 動画アップロード・公開 | パイプラインステータス表示 |
| UC-011 | 生成失敗時のリトライ | エラー・リトライUI |

## 1. 競合・類似ダッシュボードの UX 分析

### 1.1 現行プラットフォームの問題点（AI Theater 視点）

| # | プラットフォーム / ツール | 問題 | AI Theater での影響 |
|---|------------------------|------|---------------------|
| 1 | Runway Dashboard | エディタ機能（プロンプト入力・パラメータ調整）が主体で、バッチ生成・キュー管理の概念がない | 月30本の自動生成を管理するにはバッチ/キュー視点の UI が必須 |
| 2 | Bull Board (BullMQ) | ジョブの技術的ステータス（waiting/active/completed/failed）のみ表示で、ビジネスコンテキスト（コスト・チャンネル・品質）が欠如 | 運用者は「何のために」ジョブが動いているかを把握できない |
| 3 | Taskforce.sh | プロフェッショナルだが月額課金（$49+）で個人開発の $50/月 予算に不適 | 自前のライトウェイト管理UIが必要 |
| 4 | CI/CD ダッシュボード (GitHub Actions等) | パイプラインの段階表示は優れているが、メディアプレビュー機能がない | 生成結果を即座にプレビューできる必要がある |
| 5 | Vercel Dashboard | デプロイステータスの段階表示・リアルタイムログは参考になるが、コスト表示がない | 動画生成コストの可視化が AI Theater の最重要機能 |

### 1.2 採用すべきパターン

| パターン元 | パターン | AI Theater への適用 | 優先度 |
|-----------|---------|-------------------|--------|
| **GitHub Actions** | パイプライン段階表示（ステップごとのステータスアイコン） | PipelineStageIndicator として採用 | P0 |
| **Vercel** | リアルタイムログストリーム | 生成ログのライブ表示（MVP外、Phase 2） | P2 |
| **Bull Board** | ジョブステータスフィルタ（waiting/active/completed/failed） | JobStatusFilter として採用 | P0 |
| **Grafana** | メトリクスカード（数値 + ミニチャート） | CostMetricCard / GenerationMetricCard として採用 | P0 |
| **Runway** | 生成結果のインラインプレビュー | GenerationPreview として採用 | P0 |
| **Smashing Magazine (Agentic AI UX)** | Delegative UI パターン（自動実行 + 人間承認ゲート） | 承認ワークフローとして採用 | P1 |

### 1.3 AI Theater の差別化機会

```
CI/CD 的アプローチ（技術者向け）
├── 表示 → ジョブID・ステータス・ログ
├── 操作 → リトライ・キャンセル
└── 監視 → エラー率・処理時間

AI Theater のアプローチ（運用者向け・コスト意識型）
├── 表示 → チャンネル名・プロンプト概要・コスト・品質スコア
├── 操作 → 承認/却下・リトライ・スケジュール変更・予算一時停止
├── 監視 → 月間コスト推移・残予算・生成本数・品質分布
└── 予防 → 予算超過アラート・品質低下アラート・API障害通知
```

## 2. ダッシュボード全体レイアウト

### 2.1 ページ構成

| パス | ページ名 | 認証 | 説明 |
|------|---------|------|------|
| `/admin/pipeline` | パイプラインダッシュボード | **admin必須** | メイン管理画面 |
| `/admin/pipeline/jobs/[jobId]` | ジョブ詳細 | **admin必須** | 個別ジョブの詳細・ログ・プレビュー |
| `/admin/pipeline/templates` | テンプレート管理 | **admin必須** | プロンプトテンプレートの CRUD |
| `/admin/pipeline/schedule` | スケジュール管理 | **admin必須** | 生成スケジュールの設定 |
| `/admin/pipeline/cost` | コスト管理 | **admin必須** | 月間コスト詳細・予算設定 |

**認証方式**: Clerk の `organizationRole` または `publicMetadata.role === "admin"` で制御。
一般視聴者は `/admin/*` に一切アクセスできない。

## 3. コンポーネント設計

### 3.1 MetricCard（メトリクスカード）

> Grafana / Vercel 風の数値 + ミニ進捗インジケータ

```
┌─────────────────────┐
│ 📊 今月の生成数       │
│                      │
│   12 / 30            │  ← 大きな数値 (text-3xl font-bold)
│   ▇▇▇▇▇▇▇░░░ 40%   │  ← Progress bar (shadcn/ui Progress)
│                      │
│   先月比: +3 (+33%)  │  ← 前月比 (text-xs text-secondary)
└─────────────────────┘
```

**バリエーション**:

| バリエーション | 用途 | 表示 |
|--------------|------|------|
| `generation-count` | 今月生成数 | `12/30` + Progress |
| `budget-remaining` | 残予算 | `$28.50/$50.00` + Progress (残予算20%以下で `destructive` カラー) |
| `success-rate` | 成功率 | `91.7%` + ミニ棒グラフ |
| `avg-quality` | 平均品質 | `⭐ 4.2` + Quality Score 分布ミニチャート |
| `avg-duration` | 平均生成時間 | `2m 34s` + トレンド矢印 |

**shadcn/ui コンポーネント**: `Card`, `CardHeader`, `CardTitle`, `CardContent`, `Progress`, `Badge`

**Props**:
```tsx
interface MetricCardProps {
  variant: 'generation-count' | 'budget-remaining' | 'success-rate' | 'avg-quality' | 'avg-duration'
  current: number
  max?: number            // generation-count, budget-remaining 用
  trend?: {
    value: number         // 前月比の差分
    percentage: number    // 前月比のパーセンテージ
  }
  alert?: boolean         // 閾値超過時の警告表示
}
```

> GitHub Actions のステップ表示 + Runway のプレビュー要素を融合

```
┌──────────────────────────────────────────────────┐
│ 🤖 Aurora    Runway Gen-4 Turbo    [キャンセル]   │
│                                                   │
│ "夕焼けの東京タワーと富士山のシルエット..."         │
│                                                   │
│ ──── パイプラインステージ ────────────────────    │
│ ✅ プロンプト生成  →  ✅ 画像生成  →  ✅ 動画生成  →  🔄 安全性検査  →  ○ Mux Upload  →  ○ メタデータ  →  ○ 公開 │
│                                                   │
│ [████████████████░░░░] 80%                        │
│                                                   │

```

**パイプラインステージ定義**:

| ステージ | アイコン(完了) | アイコン(実行中) | アイコン(未着手) | アイコン(失敗) |
|---------|-------------|---------------|---------------|-------------|
| プロンプト生成 | ✅ | 🔄 | ○ | ❌ |
| 画像生成 | ✅ | 🔄 | ○ | ❌ |
| 動画生成 | ✅ | 🔄 | ○ | ❌ |
| 安全性検査 | ✅ | 🔄 | ○ | ❌ |
| Mux Upload | ✅ | 🔄 | ○ | ❌ |
| メタデータ生成 | ✅ | 🔄 | ○ | ❌ |
| 公開 | ✅ | 🔄 | ○ | ❌ |

**shadcn/ui コンポーネント**: `Card`, `Badge`, `Progress`, `Button`, `Tooltip`

**Props**:
```tsx
interface ActiveJobCardProps {
  job: {
    id: string
    channelName: string
    channelAvatarUrl: string
    promptSummary: string       // プロンプトの先頭60文字
    aiModel: 'runway-gen4-turbo' | 'veo-3.1-fast'
    currentStage: PipelineStage
    stageStatuses: Record<PipelineStage, 'completed' | 'active' | 'pending' | 'failed'>
    progressPercent: number
    estimatedCost: number       // USD
    elapsedSeconds: number
    estimatedRemainingSeconds: number
  }
  onCancel: (jobId: string) => void
}

type PipelineStage =
  | 'prompt_generation'
  | 'image_generation'    // Flux Schnell で参照画像を生成（Gen-4 Turbo は Image to Video のため必須）
  | 'video_generation'
  | 'safety_check'
  | 'mux_upload'
  | 'metadata_generation'
  | 'publish'
```

**ステータスアイコン定義**:

| アイコン | ステータス | 色 | 説明 |
|---------|----------|---|------|
| ✅ | completed | `text-green-500` | 正常完了・公開済み |
| 🔄 | active | `text-blue-500` | 生成中 |
| ⏳ | waiting | `text-yellow-500` | キュー待ち |
| ❌ | failed | `text-red-500` | 失敗（リトライ可能） |
| ⚠️ | needs_review | `text-orange-500` | 安全性検査で要レビュー |
| 🚫 | rejected | `text-red-700` | 承認却下・非公開 |
| ⏸️ | paused | `text-gray-500` | 一時停止中 |

**shadcn/ui コンポーネント**: `Table`, `TableHeader`, `TableBody`, `TableRow`, `TableCell`, `Select`, `Button`, `Badge`, `Pagination`, `DropdownMenu`

## 4. エラー・リトライ時のユーザー体験設計

### 4.1 エラーステータスの分類

| エラーカテゴリ | 原因 | UI表示 | 自動リトライ | 手動操作 |
|-------------|------|--------|------------|---------|
| **API Rate Limit** | Runway/Veo の API制限 | `Badge variant="warning"` + 推定復旧時刻 | ✅ 自動（指数バックオフ） | キャンセル可 |
| **API Error** | Runway/Veo の内部エラー (5xx) | `Badge variant="destructive"` + エラーコード | ✅ 自動（3回まで） | リトライ/フォールバック切替 |
| **Safety Rejected** | 安全性検査で不合格 (< 0.80) | `Alert variant="destructive"` + 理由 | ❌ | 手動: プロンプト修正→再生成 |
| **Safety Review** | 安全性検査でグレーゾーン (0.80-0.89) | `Alert variant="warning"` + プレビュー | ❌ | 手動: 承認/却下 |
| **Quality Low** | 品質スコアが低い (< 3.0) | `Badge variant="warning"` + スコア | ❌ | 手動: 再生成/承認 |
| **Budget Exceeded** | 月額予算超過 | `Alert variant="destructive"` + 残額表示 | ❌ (全停止) | 予算追加/翌月まで待機 |
| **Mux Upload Error** | Mux へのアップロード失敗 | `Badge variant="destructive"` | ✅ 自動（3回まで） | リトライ可 |
| **Timeout** | 生成タイムアウト（10分超） | `Badge variant="destructive"` | ✅ 自動（1回） | リトライ/キャンセル |
